<objective>
Revert the 260710-keg WR-01 overcorrection that set `adaptive_icon_foreground_inset: 0`
in pubspec.yaml. Android's adaptive-icon OS-level mask crop stacks with the tool's baked-in
inset, so removing the inset entirely made the sun render nearly edge-to-edge on a flat
yellow field instead of the balanced sunrise that was already verified acceptable on a real
Samsung A25 (05-05-SUMMARY.md). Restoring the tool's default inset (16) recovers that
on-device-verified rendering.

Purpose: Close the single UAT gap (Test 1, major) blocking Play Store readiness — the
launcher icon must read as a legible sunrise, not a shrunk/crowded sun.
Output: Reverted pubspec.yaml, regenerated adaptive + legacy launcher assets at 16% inset,
regenerated store_assets/icon_512.png.
</objective>

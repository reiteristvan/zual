<objective>
De-risk the single riskiest unknown in this phase (RESEARCH Assumption A1): whether D-04's
"generate the icon programmatically in Flutter/Dart" is even possible headlessly.
`dart:ui`'s PictureRecorder/Canvas/Image.toImage APIs have NO backend outside a Flutter
engine embedder, so a plain `dart run` script cannot render them — the only device-free path
is inside `flutter test`'s Skia binding (the same mechanism golden-file tests use), with real
file I/O escaped via `tester.runAsync`. This plan proves that path works against the real
`SunrisePainter` and produces a reusable render helper, BEFORE plan 05-04 commits to
generating the full multi-layer adaptive-icon asset set. If this spike fails, plan 05-04's
fallback (a live-device screenshot of SunriseScene as the icon source) is triggered instead.

Purpose: gate PUBLISH-02's icon track on a proven render pipeline rather than an assumption.
Output: `test/tool/icon_renderer.dart` (reusable `renderPainterToPng`) and a passing spike
test that writes and validates a real PNG.
</objective>

    `image.toByteData(format: ui.ImageByteFormat.png)`, and returns

<objective>
Add scene selection to the Setup screen: a 2x2 grid of four tappable scene cards, each showing a static, theme-representative mini-preview drawn through a shared painter abstraction, with single-select behavior and a default of Shrinking disc.

Purpose: Satisfy SETUP-03 and, critically, establish the D-06 preview abstraction (`ScenePreviewPainter`) so Phase 3 can later swap in "the real scene painter rendered at progress=0" without touching any Setup-screen card/grid code.
Output: `SceneTheme` enum, `ScenePreviewPainter` abstraction + four concrete painters, `SceneGrid`/`SceneCard` widgets, and their integration into `SetupScreen`.
</objective>

## Phase Goal (MVP user story)

**As a** parent, **I want to** configure a countdown's duration and scene and launch it with my last-used settings pre-selected, **so that** my child can watch a wordless visual timer without me re-entering settings each time.

This plan adds the scene-picking slice. After it, a parent can pick both a duration and a scene before Start.

## Artifacts this phase produces (this plan's contribution)

- `SceneTheme` enum (`lib/scenes/scene_theme.dart`) — `disc | sunrise | walk | car`, the theme identity shared by Setup, persistence (Plan 04), and Phase 3.
- `ScenePreviewPainter` (`lib/scenes/scene_preview.dart`) — abstract base extending `CustomPainter`; concrete `DiscPreviewPainter`, `SunrisePreviewPainter`, `WalkPreviewPainter`, `CarPreviewPainter`.
- `SceneGrid` + `SceneCard` (`lib/widgets/scene_grid.dart`) — the 2x2 selectable card grid depending only on the abstraction.

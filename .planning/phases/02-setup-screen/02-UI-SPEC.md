# Phase 2 — UI Design Contract

> Visual and interaction contract for the Setup screen (Layout A) and its minimal placeholder
> Running screen. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Fidelity note:** This project's `CLAUDE.md` and `02-CONTEXT.md` mandate pixel-accurate
> fidelity to `design/README.md` and `design/Zual.dc.html` — colors, radii, spacing, and
> typography below are transcribed directly from those files (Layout A, the shipped default)
> and are final, not starting points. Where this contract's defaults (spacing 8pt scale,
> "3-4 type sizes / 2 weights") would normally apply, they are explicitly overridden by the
> design spec per project instruction; overrides are called out inline.

## Spacing Scale

**Exception to standard 8pt scale:** the design spec's own spacing values are final per project
fidelity mandate. They are not multiples of 4/8 and are used verbatim rather than rounded to the
nearest 8pt token.

| Token | Value | Usage |
|-------|-------|-------|
| Header padding | `52px` top / `24px` sides / `8px` bottom | Fixed header (52px accounts for status-bar safe area) |
| Body padding | `12px` top / `22px` sides / `4px` bottom | Scrollable body wrapping both sections |
| Section gap | `26px` | Vertical gap between "How long?" and "Pick a scene" sections |
| Section label margin | `0 0 14px 2px` | Space below each section label, before its grid |
| Grid gap | `12px` | Gap between cells in both the 3-col duration grid and 2×2 scene grid |
| Preset/Custom button padding | `16px 6px` | Internal padding of each of the 6 duration-grid buttons |
| Scene card padding | `10px` | Internal padding of each of the 4 scene-grid cards |
| Custom stepper row padding | `14px`, `margin-top: 14px` | Padding/offset of the revealed stepper row |
| Custom stepper internal gap | `20px` | Gap between `−`, value block, `+` |
| Footer padding | `14px` top / `22px` sides / `26px` bottom | Fixed footer wrapping the Start button |
| Start button padding | `20px` | Internal padding of the Start button |
| Stepper button size | `48px` diameter (circular) | `−` / `+` buttons |
| Scene mini-preview height | `74px` | Height of each scene card's preview box |

Exceptions: entire scale above is an intentional exception to the standard 8pt default — locked
by `design/README.md` fidelity requirement, not to be rounded or substituted.

## Interaction Contract

### Duration selection
- Tapping a preset button (1/5/10/15/30) or "Custom" selects it immediately; selection draws
  the 3px `#7FA87A` ring overlay described above. Selecting a preset while Custom is open closes
  (hides) the Custom stepper row. Selecting "Custom" reveals the stepper row below the grid
  (grid itself does not reflow/resize).
- **Custom stepper hold-repeat (D-07/D-08):** tapping `−`/`+` changes the value by exactly 1
  minute per tap. Holding either button repeats and accelerates. Exact curve is not specified in
  `design/README.md`; the following is this contract's default (tune during implementation, but
  this is the target feel — do not ship a fixed, non-accelerating repeat rate):
  - Single tap (release before ~500ms): exactly one ±1 change, no repeat.
  - Held past ~500ms: begin repeating every ~350ms.
  - Held past ~2s: accelerate to every ~150ms.
  - Held past ~4s: floor at every ~60ms (fastest rate, held until range edge or release).
  - Repeat stops immediately at 1 (for `−`) or 120 (for `+`) regardless of hold duration.
- **Disabled edges (D-08):** the `−` button is visually disabled (greyed — reduce to ~35%
  opacity on both the `#F1E6D3` fill and glyph) and non-interactive at `customMin == 1`; the `+`
  button likewise disables at `customMin == 120`. Disabled buttons do not respond to tap or hold,
  and should be excluded from focus/tap-target semantics while disabled.
- `customMin` starts at its design-spec default of **3** the first time the Custom row is shown
  in a given screen session (matches `design/README.md`'s `customMin: 3` default), lives only in
  the Setup screen's local widget state, and is **not** reset by toggling the Custom row closed
  and reopened within the same app session — only an app restart returns it to the default 3
  (since custom values are never persisted, per D-10).

### Scene selection
- Tapping any of the 4 scene cards selects it immediately (single-select, no multi-select);
  selection draws the same 3px ring overlay. No animation on the mini-previews themselves in
  this phase (D-05 — static only; Phase 3 owns animation).

### Start
- Always enabled (a duration and theme are always selected — there is no "nothing chosen"
  state). Tapping calls `TimerController.start(minutes)` with `showCustom ? customMin :
  durationMin`, then navigates to the placeholder Running screen.

### Persistence defaults (PERSIST-01, D-09, D-10)
- First launch (no persisted value): `durationMin = 5`, `theme = disc`, `showCustom = false`.
- Subsequent launches: restore the last-used **preset** duration and **theme** only.
- If the last-used selection before restart was Custom (not a preset), Setup falls back to the
  default preset (**5 min**) rather than attempting a "nearest preset" match — chosen for
  simplicity per D-10 ("simpler persisted state").

## Placeholder Running Screen Contract (D-01 – D-04)

This screen is this phase's minimal Start-destination stand-in. Phase 3 replaces its visual
entirely with the real Shrinking Disc scene; Phase 4 layers real Parent Controls on top. Keep it
deliberately unpolished.

| Property | Value |
|----------|-------|
| Background | `#F6EBDD` (same app background token, for visual continuity with the eventual real scene) |
| Progress indicator | A single flat-colored circle, fill `#7FA87A` (no color-zone lerp — explicitly out of scope per D-01), diameter `300px` at `remaining = 1` (matches the real Shrinking Disc's max size for continuity), `transform: scale(remaining)` where `remaining = 1 − progress`. No track ring, no drop shadow — deliberately bare (D-01: "no color zones or polish") |
| Text/numbers | None — zero text or numbers anywhere on this screen, consistent with the child-facing, wordless constraint even though this is a parent-testable stub |
| Back/escape control | Small, always-visible, top-left. Use `Icons.arrow_back`, color `#4B4038`, minimum 44×44 tap target (accessibility baseline — applies even though this project's spacing scale otherwise departs from the 8pt default). No background chip/scrim needed — plain icon is sufficient for this placeholder |
| Back control behavior | Tapping calls `TimerController.endTimer()` then returns to Setup — **immediately, no confirmation dialog** (D-02). This is not the real Phase 4 Parent Controls UX (no 850ms long-press, no bottom sheet, no pause/resume) — explicitly out of scope per D-03 |
| Completion behavior | When `TimerController.phase` reaches `TimerPhase.done`, auto-navigate back to Setup. No chime, no "All done" pill (Phase 4 scope) — just close the loop (D-04) |
| Gestures | No long-press gesture recognizer wired on this screen at all (D-03) |

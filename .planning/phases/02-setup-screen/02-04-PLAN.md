<objective>
Make the Setup screen remember the last-used preset duration and theme across launches, pre-selecting them before the first frame, with defensive read-side validation of the stored values.

Purpose: Satisfy PERSIST-01 and the Tampering control (validate restored preferences on read, not just on write), using shared_preferences read once in main() before runApp to avoid a default-value flash.
Output: `SetupPreferences` load/persist wrapper + test, shared_preferences added to pubspec, main() preload wiring, and Start-time persistence in SetupScreen.
</objective>

## Phase Goal (MVP user story)

**As a** parent, **I want to** configure a countdown's duration and scene and launch it with my last-used settings pre-selected, **so that** my child can watch a wordless visual timer without me re-entering settings each time.

This plan delivers the "pre-selected last-used settings" half of the story.

- shared_preferences dependency (pubspec.yaml).
- main() preload sequence; `MyApp`/`SetupScreen` gain `initialTheme` alongside `initialDurationMin`.

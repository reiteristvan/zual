<objective>
Fix the carried-forward Phase 3 defect (D-10, STATE.md Blockers): `SceneRendererState`'s
decorative loops (star twinkle, character/car bob, wheel spin) reset to phase 0 every time the
`Ticker` stops and restarts, because the Ticker's `elapsed` argument restarts from zero on each
`start()`. Pause/Resume (wired for the first time in Plan 04-04) is exactly what triggers this snap.
The fix accumulates an offset across ticker segments so loops resume seamlessly.

**Why this serves CTRL-02:** Pause/Resume is a Parent Controls action (CTRL-02); the paused‚Üíresumed
scene must read as calm and continuous rather than glitching, which is only reachable once this fix
lands.

Purpose: Preserve the "timer paused = scene frozen, resume = continues" mental model from Phase 3.
Output: A minimal single-file offset accumulation in `scene_renderer.dart` + a continuity test.
</objective>

# Phase 4 — UI Design Contract

> Visual and interaction contract for the Parent Controls bottom sheet, the mute toggle, and
> the Completed state (chime + settled scene + breathing "All done" pill) on
> `lib/screens/running_screen.dart`. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Fidelity note:** Per this project's `CLAUDE.md` and `04-CONTEXT.md`, colors, radii, and
> timings transcribed from `design/README.md` §§G–H and "Interactions & Behavior" are **final**,
> not starting points. The mute toggle (D-01–D-04) is the one UI element added beyond the
> literal design doc (REQUIREMENTS.md's CTRL-02 extends §G) — its icon/copy are locked by
> `04-CONTEXT.md`, but its exact padding/placement within the sheet header is this contract's
> call, marked as a default below. Where a value is genuinely unspecified by any upstream
> source (grab handle size, pill offset from bottom, sheet content padding), it is called out
> explicitly as a **default (Claude's discretion)** rather than presented as if it were locked.
>
> **Scope boundary:** this phase deletes the Phase-3 scaffolding visible back `IconButton` on
> `RunningScreen` (lines 79–92) outright — it is replaced, not kept as a fallback. All gesture
> detection, sheet presentation, chime triggering, and pill rendering live in `RunningScreen`
> (the composition root), never inside a scene widget, preserving the SCENE-05 boundary already
> established in `03-UI-SPEC.md`.

## Parent Controls Sheet Contract

**Trigger (CTRL-01):** a hidden `RawGestureDetector` + `LongPressGestureRecognizer(duration:
Duration(milliseconds: 850))` spans the entire running screen (`HitTestBehavior.opaque`),
layered above the scene but never inside it. **D-08 (locked):** the hold is fully silent and
invisible until it fires — no scrim, ripple, darkening, or any other build-up affordance during
the 850ms. Releasing before the threshold does nothing at all (no partial-state visual). Use the
SDK's default `postAcceptSlopTolerance` — no custom drift tolerance (per `04-CONTEXT.md`
Claude's Discretion, confirmed).

**Presentation:** `showModalBottomSheet(backgroundColor: Colors.transparent, barrierColor:
Color(0x6B28201A), builder: (ctx) => BackdropFilter(filter: ImageFilter.blur(sigmaX: 3, sigmaY:
3), child: /* sheet */))`. The sheet content itself is a `Container` with `color: #FBF4E8`,
`borderRadius: BorderRadius.vertical(top: Radius.circular(30))`, and the `sheet` shadow token
(`0 -10px 40px rgba(0,0,0,0.25)`) from Design Tokens.

**Layout order, top to bottom** (locked structure per §G, mute icon placement per D-01):

1. Grab handle (centered, decorative, see Spacing Scale for size/color)
2. Header row: `"Parent controls"` title (left-aligned) + mute icon button (right-aligned, same row) — D-01's "small icon button in the sheet header, near the grab handle/title"
3. Pause/Resume button (full-width, `#7FA87A` fill, 22px radius)
4. End timer button (full-width, `#E0805F` fill, 22px radius)
5. "Keep watching" text button (centered, no fill, tertiary)

**Known rendering caveat (carried from `04-RESEARCH.md` Pitfall 3):** `BackdropFilter` behind a
`showModalBottomSheet` has documented jank/flicker issues on some Flutter/Impeller
configurations. Verify the blurred scrim on a real Android device (mirrors this project's
established Phase 3 D-03 smoothness-check precedent) before treating the 3px blur as
unconditionally final. **Pre-agreed fallback if jank appears:** drop to a flat, non-blurred
scrim at the same `rgba(40,32,26,0.42)` color — do not silently ship a different color/opacity
instead, and flag the substitution to the human rather than downgrading unannounced.

## Completion State Contract

**Scene settle (CTRL-03, already satisfied by Phase 3):** at `progress = 1`, every scene is
already visually correct with no code change needed here — disc at its `0.001` floor (pure
red), sunrise fully daylit, character/car at 68% (visually at the house). This phase's job is
layering the chime + pill on top of that already-calm frame, not re-touching scene rendering.

**Chime (CTRL-03, D-05/D-06/D-07) — non-visual, retained here for completeness:**
- Two sine tones: **D5 (587.33 Hz)** then **G5 (783.99 Hz)**, tone starts ~0.3s apart
- Envelope per note: gain ramps to ~0.16 over 60ms, then exponential decay to ~0 over ~1.1s
- No harsh alarm, no looping, no celebration sound — calm-only, per project's Out-of-Scope table
- Plays through the standard media/notification channel (`USAGE_MEDIA`/`STREAM_MUSIC` on
  Android) — respects the device's **media volume slider**. **Product-facing nuance (verify
  against, do not treat as a bug):** Android's ringer/DND "silent mode" does **not** silence
  `STREAM_MUSIC`; the actual "make it quiet" mechanism on Android is this phase's own in-app
  mute toggle (D-01/D-02), not the phone's silent switch. UAT/verification steps should test
  against media volume, not ringer silent mode.
- Fires exactly once per completion, including on first foreground-reveal if the app was
  backgrounded through `done` (D-07) — must not replay on subsequent rebuilds while parked in
  `done`
- Skipped entirely when `soundOn == false`
- Exact envelope/decay constants are an implementation detail to tune by ear against this
  description — not a locked numeric formula (per `04-RESEARCH.md` Open Question 1); a human
  listen-check ("does it sound calm, not alarm-like") is the appropriate verification, not a
  byte-exact assertion

**Mute toggle (CTRL-02, D-01–D-04):**
- Icon-only button, `Icons.volume_up` (unmuted) / `Icons.volume_off` (muted), placed per the
  Sheet Contract's header row above
- Persists via `shared_preferences`, extending the existing `SetupPreferences` load/validate
  pattern (a corrupted/wrong-typed stored value falls back to the D-04 default, `true`/unmuted
  — same tamper-defense precedent as `durationMin`/`theme`)
- Default on first launch (no persisted value): **unmuted**

**"All done" pill (CTRL-04):**
- Text: `"All done — tap when ready"`, appears once `TimerPhase.done` is reached
- Breathing loop: `scale 1 → 1.05 → 1`, `2.8s`, ease-in-out, infinite (`AnimationController` +
  `CurvedAnimation`, started once and gated by visibility — see Design System note above on why
  this is the one exception to "never a second `AnimationController`")
- Positioned near the bottom of the screen, centered horizontally (exact offset defaulted in
  Spacing Scale above)
- Tapping calls `TimerController.endTimer()` then pops back to Setup — the **only** interactive
  affordance once `done` (D-09: long-press does nothing in this phase once completed)
- Child ignores it by design; only the parent is expected to tap it

**Decorative loop continuity (D-10, carried Phase 3 defect — visual quality item, not a new UI
element):** `SceneRendererState`'s decorative loops (star twinkle, character/car bob, wheel
spin) must resume from their frozen phase on Resume, not snap to phase-0. This is a rendering
fix inside `scene_renderer.dart`, not a new visual element this contract introduces — flagged
here because it directly affects whether Pause→Resume *reads* as calm and continuous (this
phase's core visual-quality bar) versus glitchy. See `04-RESEARCH.md` Pitfall 2 / Code Examples
for the concrete fix.

## Interaction Contract

- **Composition-root-only gestures (SCENE-05 boundary, carried from Phase 3):** the long-press
  recognizer, `showModalBottomSheet` call, chime trigger, and pill all live in `RunningScreen`.
  No scene widget gains its own `GestureDetector`/`AnimationController` in this phase.
- **Phase-gated availability:** the long-press → sheet gesture is live during `running` and
  `paused` only. Once `TimerPhase.done`, long-press does nothing (D-09) — the pill's single tap
  is the only remaining affordance.
- **Sheet buttons act immediately, no secondary confirms:** Pause/Resume, End timer, and Keep
  watching all take effect on a single tap (see Copywriting Contract's Destructive confirmation
  row for the explicit "no dialog" rule on End timer).
- **Mute is instantaneous and persisted:** tapping the mute icon flips `soundOn`, updates the
  icon glyph immediately, and writes through to `shared_preferences` — no separate "Save"
  step, matching the existing `SetupPreferences` fire-and-forget persistence precedent from
  Phase 2.
- **No new packages beyond `audioplayers`:** every gesture/sheet/animation technique here
  (`RawGestureDetector`, `LongPressGestureRecognizer`, `showModalBottomSheet`, `BackdropFilter`,
  `AnimationController`) is Flutter SDK-native — confirmed by `04-RESEARCH.md`'s Standard Stack.

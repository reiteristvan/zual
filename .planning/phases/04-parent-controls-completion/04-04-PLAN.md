<objective>
Replace the interim visible back button with the hidden long-press Parent Controls sheet
(CTRL-01, CTRL-02). A ~850ms silent hold opens a blurred-scrim bottom sheet offering
Pause/Resume, End timer, Keep watching, and a header mute toggle — all wired to the existing
`TimerController` and the `SetupPreferences.persistSoundOn` path from Plan 04-02. This also
threads the injected `ChimePlayer` and a shared mute `ValueNotifier<bool>` through the widget
tree so Plan 04-05 can fire the chime.

**Phase user story (MVP):** As a parent, I want to discreetly control a running timer via a
hidden long-press, so that I can pause, end, or mute it without a visible control my child
could reach or accidentally trigger. (Derived from the ROADMAP Phase 4 outcome goal.)

Purpose: Deliver the parent-only control surface as a single vertical slice — gesture → sheet →
controller actions → persisted mute — matching design/README.md §G and 04-UI-SPEC.md exactly.
Output: Reworked `RunningScreen` + new tokens + threaded dependencies + widget-test harness.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| child touch → controls | The running screen is child-facing; the controls surface must NOT be reachable by a normal tap |
| UI → local storage | Mute toggle writes an untrusted-on-next-read value to shared_preferences |
| UI → domain | Sheet buttons invoke TimerController state transitions |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-04 | Tampering (UX boundary) | Long-press recognizer | medium | mitigate | 850ms sustained threshold + D-08 fully-silent/invisible hold (no build-up affordance) makes accidental child discovery unlikely; verified by the "shorter press does nothing" test |
| T-04-05 | Denial of Service (parent control) | RawGestureDetector reliability | low | mitigate | Use SDK default LongPressGestureRecognizer slop tolerance (no custom postAcceptSlopTolerance) so a parent's minor finger drift still fires; HitTestBehavior.opaque spans the whole screen |
| T-04-03 | Tampering | persistSoundOn write / soundOn read | medium | mitigate | Write-through is fire-and-forget; the read-side validate/fallback (Plan 04-02) is the durable defense against a corrupted stored value |
</threat_model>

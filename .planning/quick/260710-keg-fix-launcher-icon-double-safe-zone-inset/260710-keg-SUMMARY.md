# Quick Task 260710-keg: Fix Launcher Icon Double Safe-Zone Inset Summary

**Removed the stacked 16% flutter_launcher_icons inset on top of the art's own baked-in safe-zone padding, and added a flattened 512x512 RGBA store-listing icon at `store_assets/icon_512.png`.**

## Accomplishments
- Fixed WR-01: `adaptive_icon_foreground_inset: 0` added to the `flutter_launcher_icons` config in `pubspec.yaml`, and Android assets regenerated via `dart run flutter_launcher_icons` — `ic_launcher.xml` now reads `android:inset="0%"` (previously `16%`), so the on-device sun disc renders at its intended ~64% diameter instead of ~43%.
- Added `test/tool/generate_store_icon_test.dart`, a `flutter test` that composites the committed `icon_background.png` + `icon_foreground.png` onto a single 512x512 canvas via headless `dart:ui` rasterization, producing `store_assets/icon_512.png` — a flat RGBA PNG for the Play Console "App icon" / hi-res store-listing upload requirement.
- `flutter build apk --debug` succeeds against the regenerated resources; full `flutter test` suite (133 tests) passes with no regressions.

## Decisions Made
- Set the tool's inset to 0 rather than touching `icon_painters.dart`'s art (scope guardrail in the plan explicitly forbade modifying the painter or source PNGs) — the safe-zone margin already exists in the art itself via the 0.32 sun-disc radius fraction.
- Generated the store icon inside a `flutter test` (not a plain `dart run` script) because `dart:ui`'s `PictureRecorder`/`Image` rasterization APIs have no headless backend outside the Flutter test engine's Skia binding, matching the existing `icon_renderer.dart` approach.

<objective>
Make the app's Android build identity real and production-signed: replace the
`com.example.zual` placeholder applicationId with `com.ireiter.zual` (D-05), give the
Android launcher a real display label (D-06), bump the version/description out of scaffold
defaults, and wire a production `release` signing config that reads a gitignored

developer can produce a signed, real-identity release bundle that Play Console will accept
(debug-signed bundles are rejected).

with the debug key and carries a `com.example.*` identity.
Output: Edited `build.gradle.kts` (identity + signing), `AndroidManifest.xml` label,
`pubspec.yaml` version/description; a developer-generated keystore + `key.properties`; a
verified signed `.aab`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local repo -> git remote | Signing credentials (keystore, passwords) must not cross this boundary |
| build config -> Play Store identity | applicationId chosen here becomes the permanent published identity |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01 | Information Disclosure | upload-keystore.jks / android/key.properties | high | mitigate | Both created only at the human checkpoint (Task 2), never by Claude; checkpoint step 3 hard-gates on `git status` showing them ignored. Covered by existing patterns: android/.gitignore (`key.properties`, `**/*.keystore`, `**/*.jks`) and root .gitignore (`**/android/key.properties`, `*.jks`). Chosen keystore name `upload-keystore.jks` matches `*.jks`. |
| T-05-02 | Spoofing | applicationId | medium | mitigate | Use the developer-account-scoped `com.ireiter.zual` (D-05) rather than a generic/example ID, reducing impersonation surface on Play Store. |
| T-05-03 | Tampering | debug-signed release bundle | high | mitigate | Task 1 removes debug signing from the release buildType whenever key.properties exists; Task 3 asserts the built .aab does not carry `CN=Android Debug`. |
</threat_model>

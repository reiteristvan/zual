<objective>
Add the audio foundation for the completion chime (CTRL-03): a pure-Dart two-tone WAV
synthesizer, a plugin-free ChimePlayer interface with a no-op default, and a single
`audioplayers`-backed adapter. No UI wiring here — this plan produces the reusable audio
primitives that Plan 04-05 triggers on timer completion.

**Phase user story (MVP):** As a parent, I want a running timer's completion to resolve into
a calm finished state with a soft chime, so that my child is gently signaled without an
overstimulating alarm. (Derived from ROADMAP Phase 4 goal, which is stated as an outcome.)

Purpose: Isolate all audio/platform concerns behind a testable interface following the
existing ScreenWake/WakelockScreenWake precedent, so the chime is unit-testable in pure Dart
and widget tests never hit `MissingPluginException`.
Output: `lib/audio/` package (3 files) + a pure-Dart unit test + the `audioplayers` dependency.
</objective>

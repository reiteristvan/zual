<objective>
Extend the existing `SetupPreferences` value object with a persisted `soundOn` boolean so the
parent's mute choice survives app restarts (CTRL-02 mute persistence, D-02/D-04). This reuses
`shared_preferences` — no second persistence mechanism — and applies the same validate-on-every-read
tamper defense already protecting `durationMin`/`theme` (threat T-02-02 precedent, V5).

Purpose: Give the mute toggle a durable home before Plan 04-04 wires the UI, keeping all
persistence logic in one place following Phase 2's precedent.
Output: `soundOn` field + loader fallback + `persistSoundOn` writer + tampering/default unit tests.
</objective>
